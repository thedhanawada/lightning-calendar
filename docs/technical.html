Technical Documentation

forceCalendar Core is a high-performance calendar computation engine implemented in pure JavaScript ES6+ modules. This documentation covers the technical architecture, algorithms, and implementation details.

System Architecture

The engine follows a modular, event-driven architecture with clear separation of concerns. Each module is self-contained with well-defined interfaces.

Spatial Indexing Algorithm

The core innovation is our R-tree based spatial index that treats time as a spatial dimension, enabling O(1) average-case lookups regardless of dataset size.

Implementation

class SpatialIndex {
    constructor() {
        this.tree = new RTree();
        this.index = new Map();
    }

    insert(event) {
        // Convert time to spatial coordinates
        const bounds = {
            minX: event.start.getTime(),
            maxX: event.end.getTime(),
            minY: 0,
            maxY: 1
        };

        // Insert into R-tree
        this.tree.insert(bounds, event.id);

        // Maintain hash index for O(1) ID lookups
        this.index.set(event.id, event);
    }

    query(start, end) {
        // Range query in logarithmic time
        const results = this.tree.search({
            minX: start.getTime(),
            maxX: end.getTime(),
            minY: 0,
            maxY: 1
        });

        return results.map(id => this.index.get(id));
    }
}

API Reference

Calendar Class

new Calendar(config) Returns: Calendar instance

Creates a new calendar instance with the specified configuration.

config.timeZone String IANA timezone identifier (default: system timezone)
config.weekStartsOn Number Day week starts on (0=Sunday, 1=Monday, etc.)
config.locale String Locale for date formatting (default: en-US)
calendar.addEvent(eventData) Returns: Event

Adds a new event to the calendar.

eventData.id String Unique identifier for the event
eventData.title String Event title
eventData.start Date Event start date/time
eventData.end Date Event end date/time (optional)
eventData.recurrence String RFC 5545 RRULE string (optional)
calendar.getEventsForDay(date) Returns: Array<Event>

Returns all events for the specified date.

date Date The date to query events for
calendar.detectConflicts(event) Returns: Array<Event>

Finds all events that conflict with the given event.

event Event Event to check for conflicts

Timezone Handling

Full IANA timezone database support with automatic DST handling and historical timezone data. 

// Timezone conversion example
const calendar = new Calendar({ timeZone: 'America/New_York' });

// Event created in New York time
const event = calendar.addEvent({
    title: 'Meeting',
    start: new Date('2025-01-15T14:00:00'), // 2 PM EST
    timeZone: 'America/New_York'
});

// Convert to Tokyo time
const tokyoTime = calendar.convertTimezone(event, 'Asia/Tokyo');
// Result: 2025-01-16T04:00:00 (4 AM JST next day)

// Handles DST transitions automatically
const dstEvent = calendar.addEvent({
    title: 'DST Test',
    start: new Date('2025-03-09T01:30:00'), // During DST transition
    timeZone: 'America/New_York'
});

Supported Features

  • 387 IANA timezone identifiers
  • Automatic DST transitions
  • Historical timezone data (1970-2050)
  • Timezone abbreviation handling (EST, PST, etc.)
  • UTC offset calculations

Recurrence Engine

Full RFC 5545 (iCalendar) RRULE implementation supporting complex recurrence patterns.

Supported Recurrence Rules

// Daily recurrence
'FREQ=DAILY;COUNT=10'

// Weekdays only
'FREQ=DAILY;BYDAY=MO,TU,WE,TH,FR'

// Every other week on Tuesday and Thursday
'FREQ=WEEKLY;INTERVAL=2;BYDAY=TU,TH'

// First Monday of every month
'FREQ=MONTHLY;BYDAY=1MO'

// Last weekday of every month
'FREQ=MONTHLY;BYDAY=MO,TU,WE,TH,FR;BYSETPOS=-1'

// Yearly on January 1st and July 4th
'FREQ=YEARLY;BYMONTH=1,7;BYMONTHDAY=1,4'

// Complex: Every 3rd Thursday of November
'FREQ=YEARLY;BYMONTH=11;BYDAY=3TH'

Performance Optimizations

  • Lazy evaluation - instances generated on demand
  • Bounded expansion - limits instances to visible range
  • Instance caching - reuses calculated occurrences
  • Fast path optimization for simple patterns

Memory Management

Adaptive memory management system that automatically adjusts to available resources.

LRU Cache Implementation

class LRUCache {
    constructor(maxSize = 1000) {
        this.maxSize = maxSize;
        this.cache = new Map();
    }

    get(key) {
        if (!this.cache.has(key)) return undefined;

        // Move to end (most recently used)
        const value = this.cache.get(key);
        this.cache.delete(key);
        this.cache.set(key, value);
        return value;
    }

    set(key, value) {
        // Remove oldest if at capacity
        if (this.cache.size >= this.maxSize) {
            const firstKey = this.cache.keys().next().value;
            this.cache.delete(firstKey);
        }

        this.cache.set(key, value);
    }
}

Memory Pressure Detection

The engine monitors memory usage and automatically adjusts cache sizes and garbage collection frequency based on available memory.

Integration Examples

Salesforce Lightning Web Component

import { LightningElement } from 'lwc';
import { Calendar } from '@forcecalendar/core';

export default class CalendarComponent extends LightningElement {
    calendar;

    connectedCallback() {
        // Initialize calendar - Locker Service compatible
        this.calendar = new Calendar({
            timeZone: 'America/Los_Angeles'
        });

        // Load events from Salesforce
        this.loadEvents();
    }

    async loadEvents() {
        const events = await getEvents(); // Apex call
        events.forEach(event => {
            this.calendar.addEvent({
                id: event.Id,
                title: event.Subject,
                start: new Date(event.StartDateTime),
                end: new Date(event.EndDateTime)
            });
        });
    }
}

React Integration

import { useState, useEffect } from 'react';
import { Calendar } from '@forcecalendar/core';

function useCalendar(config) {
    const [calendar] = useState(() => new Calendar(config));
    const [events, setEvents] = useState([]);

    useEffect(() => {
        const handler = ({ detail }) => {
            setEvents(detail.events);
        };

        calendar.on('eventsChange', handler);
        return () => calendar.off('eventsChange', handler);
    }, [calendar]);

    return { calendar, events };
}

// Usage
function CalendarApp() {
    const { calendar, events } = useCalendar({
        timeZone: 'Europe/London'
    });

    return ;
}

Node.js Backend

import { Calendar } from '@forcecalendar/core';
import express from 'express';

const app = express();
const calendar = new Calendar();

app.get('/api/events/:date', (req, res) => {
    const date = new Date(req.params.date);
    const events = calendar.getEventsForDay(date);

    res.json(events);
});

app.post('/api/events', (req, res) => {
    const event = calendar.addEvent(req.body);

    // Check for conflicts
    const conflicts = calendar.detectConflicts(event);
    if (conflicts.length > 0) {
        return res.status(409).json({ conflicts });
    }

    res.json(event);
});
